Gas Saver API - Development Plan & Execution
1. Project Overview
Business Context
Develop a Node.js TypeScript API service that enables USDT (TRC-20) holders to convert their deposits into TRON ENERGY credits, eliminating the need to purchase TRX for network fees.
Core Objectives
* Process USDT deposits and convert to ENERGY credits
* Provide secure user authentication and management
* Enable real-time ENERGY delegation and monitoring
* Support admin operations for system management
* Achieve 99.9% uptime with p95 latency < 300ms
2. Technical Requirements
Technology Stack
* Runtime: Node.js 18 LTS
* Language: TypeScript 5
* Framework: Express 5 + express-async-errors
* Database: PostgreSQL 15 (AWS RDS) + Prisma ORM
* Cache/Queue: Redis 7 + BullMQ 3
* Blockchain: TronWeb SDK + TronGrid WebSocket
* Validation: Zod + openapi-zod-converter
* Logging: Winston-pino (JSON to CloudWatch)
* Security: Argon2id, TOTP 2FA, Helmet
* Testing: Vitest + Supertest
Performance Targets
* p95 REST latency < 300ms
* Support 100 deposits/minute
* 99.9% monthly uptime SLA
* Real-time deposit detection (≤15s)
