Node.js TypeScript Project Structure Guide
A comprehensive guide for organizing Node.js TypeScript projects following industry best practices and clean architecture principles.
Overview
This guide establishes a standardized project structure for Node.js TypeScript applications that promotes maintainability, scalability, and developer productivity. The structure follows domain-driven design principles with clear separation of concerns.
Core Principles
1. Modular Architecture
* Organize code by business domains/features rather than technical layers
* Each module should be self-contained and loosely coupled
* Clear boundaries between different business concerns
2. Clean Architecture
* Separate business logic from infrastructure concerns
* Dependency inversion (high-level modules don't depend on low-level modules)
* Framework-agnostic core business logic
3. Type Safety
* Strict TypeScript configuration
* Comprehensive type definitions
* Runtime validation where needed
4. Scalability
* Structure supports growth from small to large applications
* Easy to add new features without disrupting existing code
* Clear patterns for team collaboration

Module Structure Guidelines
Standard Module Files
Each module should follow this consistent structure:
1. Controller (*.controller.ts)
* Handles HTTP requests and responses
* Input validation and sanitization
* Calls appropriate service methods
* Returns formatted responses
// Example: user.controller.ts
export class UserController {
  constructor(private userService: UserService) {}


  async createUser(req: Request, res: Response) {
    // Handle request, call service, return response
  }


  async getUser(req: Request, res: Response) {
    // Implementation
  }
}
2. Service (*.service.ts)
* Contains business logic
* Orchestrates data operations
* Handles business rules and validations
* Framework-agnostic
// Example: user.service.ts
export class UserService {
  constructor(private userRepository: UserRepository) {}


  async createUser(userData: CreateUserDto): Promise<User> {
    // Business logic implementation
  }
}
3. Repository (*.repository.ts)
* Data access layer
* Database queries and operations
* Abstracts database implementation details
// Example: user.repository.ts
export class UserRepository {
  async create(userData: CreateUserData): Promise<User> {
    // Database operations
  }


  async findById(id: string): Promise<User | null> {
    // Database query
  }
}
4. Routes (*.routes.ts)
* Route definitions
* Middleware application
* Route-level validation
// Example: user.routes.ts
const router = express.Router();


router.post('/users', validateCreateUser, userController.createUser);
router.get('/users/:id', authMiddleware, userController.getUser);


export { router as userRoutes };
5. Types (*.types.ts)
* TypeScript interfaces and types
* DTOs (Data Transfer Objects)
* Request/response types
// Example: user.types.ts
export interface User {
  id: string;
  email: string;
  name: string;
  createdAt: Date;
}


export interface CreateUserDto {
  email: string;
  name: string;
  password: string;
}
6. Validation (*.validation.ts)
* Input validation schemas
* Validation middleware
* Custom validators

Best Practices
1. Naming Conventions
* Files: Use kebab-case or camelCase consistently
   * user.controller.ts, auth.service.ts
* Classes: Use PascalCase
   * UserController, AuthService
* Functions/Variables: Use camelCase
   * createUser, isAuthenticated
* Constants: Use SCREAMING_SNAKE_CASE
   * API_BASE_URL, DEFAULT_PAGE_SIZE
2. Import/Export Patterns
// Barrel exports in index.ts files
export { UserController } from './user.controller';
export { UserService } from './user.service';
export { UserRepository } from './user.repository';
export * from './user.types';


// Clean imports in consuming files
import { UserController, UserService, CreateUserDto } from '../user';
3. Error Handling
// shared/exceptions/base.exception.ts
export abstract class BaseException extends Error {
  abstract statusCode: number;
  abstract status: string;
  
  constructor(message: string) {
    super(message);
    Error.captureStackTrace(this, this.constructor);
  }
}


// shared/exceptions/validation.exception.ts
export class ValidationException extends BaseException {
  statusCode = 400;
  status = 'fail';
  
  constructor(message: string) {
    super(message);
  }
}
4. Dependency Injection
Consider using a DI container or manual dependency injection:
// Manual DI example
export class UserModule {
  static create() {
    const userRepository = new UserRepository();
    const userService = new UserService(userRepository);
    const userController = new UserController(userService);
    
    return {
      controller: userController,
      service: userService,
      repository: userRepository,
    };
  }
}
